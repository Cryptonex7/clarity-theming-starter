Starting in Clarity 3.0, Clarity's rem-based sizing changed from 24px to 20px. Please use either the rem-sizing variables in Clarity or multiply any exising rem values relying on the previous base size of 24px by 1.2.

Rem Units in Clarity 3+

Leaving rem units in your application that used to rely on the 24px base sizing of Clarity will result in smaller spacing and sizes.

0.5rem was 12px in Clarity 2.x.

0.5rem is equivalent to 10px in Clarity 3.x and up.

The recommended fix is replacing set rem units in your application with Clarity's remSize variables.

These can be found in /node_modules/@clr/ui/src/utils/_variables.global.scss.

To import them you will also need to import the color variables from /node_modules/@clr/ui/src/color/_variables.color.scss.

padding: 0.25rem 0.5rem;
margin: 0.75rem;
font-size: 0.416667rem; // equals 10px

...becomes...

@import '~@clr/ui/src/color/variables.color';
@import '~@clr/ui/src/utils/variables.global';
padding: $clr_baselineRem_0_25 $clr_baselineRem_0_5;
margin: $clr_baselineRem_0_75;
font-size: $clr_baselineRem_10px; // we have some px values defined for convenience

Alternately, you could implement your own fix by multiplying existing rem units by 1.2.

You could create a mixin that does this for you, similarly to how Clarity remSizing variables are implemented. Or just update your values in place.

padding: 0.25rem 0.5rem;
margin: 0.75rem;
font-size: 0.416667rem; // equals 10px

...becomes...

@function updateRem($size: 1rem) {{ '{' }} return $size * 1.2; {{ '}' }} padding: updateRem(0.25rem) updateRem(0.5rem);
margin: 0.9rem; // equals 1.2 * 0.75rem
font-size: 1.2 * 0.416667rem;
